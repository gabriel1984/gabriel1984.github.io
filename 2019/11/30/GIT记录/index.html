阅读量:

我想把hexo博客从github迁移到阿里云的centos服务器，顺便在centos上搭建一个git服务器。

创建新用户

ssh连接centos服务器后，先添加一个专用于git版本控制的用户

$ adduser git
$ passwd  git 
#修改用户密码，权限暂时不用改

添加ssh密钥用于免密登陆

使用ssh-key本地生成密钥
使用scp将生成的pub公钥传送到目标主机
使用cat 和 >> 命令将pub内容追加到 .ssh/authorized_keys

安装git

方法1. yum -y install git
方法2. 编程git源码
如果采用方法1，会安装yum源中比较旧的git，当然也可以尝试更换yum源，寻找较新版本的git
方法2是一种通用的方法，还可以指定git的安装目录。下面讲一下如何编译安装git以及配置git。

$ wget https://github.com/git/git/archive/v2.24.0.tar.gz 
#在github上查找git的源码版本

$ tar -zxf git-2.0.0.tar.gz 
#解压源码

$ cd git-2.0.0
$ make configure 
$ ./configure --prefix=/usr 
#设定git的编译目录，可以更改

$ make all doc info 
#编译

$ sudo make install install-doc install-html install-info

以上就是完整的编译方式

$ git clone git://git.kernel.org/pub/scm/git/git.git 
#更新git的方式

如果指定的目录不在环境变量中，那么需要在用户的配置文件中添加git的环境变量，比如 写入/etc/profile 文件，才可以使得git命令生效。

测试git服务

如果是多用户使用git服务器，可以建立bare仓库，该仓库不包含用户所使用的工作文件，只包含仓库的版本提交记录。使用git init –bare 初始化仓库

如果是采用的是 git init 命令初始化，将会在仓库下生成.git文件，这跟用户所使用的本地仓库类似。

远程仓库的命令执行:

$ mkdir test.git 
#注意命名习惯

$ cd test.git
$ git init

本地仓库的命令执行:

$ cd sourcefile
$ git init 
$ git add .  
#添加所以文件到工作区

$ git commit -m 'first commit'  
#提交

$ git remote add origin git@ip:/path/test.git 
#path 是test.git的pwd路径

$ git push -u origin master  
#推送到远程仓库。如果遇到提示无权限推送，需要在远程centos主机下执行如下命令

$ cd test.git
$ vim config

在文件的末尾添加如下代码

[receive]
denyCurrentBranch = ignore

其他常用命令

$ git branch dev 
#建立dev分支
$ git checkout dev 
#切换到dev分支
$ git branch -a 
#查看所有分支提交情形
$ git status  
#查看工作区的状态

到此为止，git服务器搭建完成。如果是多人登陆git，为了安全着想，可以在远程主机上限制采用git用户名登陆的shell权限：

  1. 在git用户目录下添加 git-command-shell文件夹
  2. 编辑 /etc/passwd 文件，将git对应的bash改成 git-shell
