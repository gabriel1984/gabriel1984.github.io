    今年英雄联盟世界总决赛比赛期间，b站直播间气氛火热，弹幕铺天盖地，可恨自己手速不够快，刷出的”777”根本不够看。于是想到了做一个弹幕发射机，实现自动化发射弹幕。

    实现思路

    总体上讲有两种不同的实现方式。第一种方式使用python中的requests库自动发送预设的文本，模拟输入和点击，第二种方式使用js脚本，加载到控制台或者油猴插件刷屏，本篇先说一下python方案的实现方式。

    运行环境

    python方案也有两种。

    第一种方式采用selenium测试平台实现模拟登陆，手动提交极验验证码或者破解极验验证码自动提交（成功率不高），然后浏览器自动跳转到对应直播间刷弹幕。
    需要的运行环境：

    python==3.7/selenium/threading/time/beautifulsoup4/requests/json/re

    第二种方案利用登陆后获取的cookie，解析得到post需要提交的参数列表，添加需要弹幕参数后发送请求。
    需要的运行环境：

    python==3.7/threading/time/requests/json/re

    建议使用虚拟环境

    代码细节

    使用selenium

    必须熟练使用Chrome等浏览器的调试工具，按F12或者右键调试都可以。

    首先构造一个封装登陆操作及参数的类

    class loginBili():
    def __init__(self):
    #构造函数
        self.url = 'https://www.bilibili.com/'
        self.curl = 'https://passport.bilibili.com/login' 
        # B站登录界面
        self.uroom = "https://live.bilibili.com/" + roomid
        #直播间链接
        self.umsg = 'https://api.live.bilibili.com/msg/send'
        #发送弹幕的api
        self.browser = webdriver.Chrome()
        #实例化一个浏览器对象
        self.browseropt = webdriver.ChromeOptions()      
        self.wait = WebDriverWait(self.browser, 20)
        # 定义显示等待，第一个参数是等待对象，第二个参数是等待时间
        self.username = USERNAME
        self.password = PASSWARD
        #用户名和密码的私有参数，由用户自己输入

        在上述参数中，发送弹幕的b站api可以在Chrome或者Charles软件中查看请求地址得到。webdriver.Chorme 是selenium的驱动库，相关驱动要到官网下载。
        还可以使用Firefox/Safari等浏览器的驱动。

    现在看一下相关参数的获取：

    username = self.wait.until(EC.presence_of_element_located((By.ID, 'login-username')))
#调用self.wait.until()是为了等待页面的加载和元素的定位，在定义时间内轮询等待操作完成。也可以采用其他等待方式，比如强制使用time.sleep()
#EC.presence_of_element_located()是定位页面元素的方式，第一个参数是定位所使用的类别，比如类名，ID等等，第二个参数是类别对应的参数值。 
username.send_keys(self.username)
#send_keys 是selenium提供的针对页面元素操作的方法，将用户提供的用户名填充到页面表单中。

    同样分两步，找到并填充发送弹幕的文本框，找到并点击发送弹幕的按钮
chat = self.wait.until(EC.presence_of_element_located((By.CLASS_NAME, 'chat-input')))

    采用requests发送弹幕：

    此方法速度相比于selenium更快，只是不够省心。

    代码细节：

    cookie = "xxxxxxxx"
#手动打开浏览器调试工具，获取当前的cookie，注意使用双引号提交
token = re.search(r'bili_jct=(.*?);', cookie).group(1)
#使用正则表达式获取请求参数中的csrf_token,七天有效
url_send = 'https://api.live.bilibili.com/msg/send'
#查找得到的弹幕发送api
headers = {
        'User-Agent': 'Mozilla/5.0 (Windows NT 6.1; WOW64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/55.0.2883.87 Safari/537.36',
        'Cookie': cookie
    }
    data = {
        'color': '16777215',
        'fontsize': '25',
        'mode': '1',
        'msg': msg,
        #想要发送的弹幕文本
        'rnd': int(time.time()),
        #时间戳
        'roomid': roomid,
        'csrf_token': token,
        #csrf验证
        'csrf': token
    }
html_send = requests.post(url_send, data=data, headers=headers)
#弹幕发送

    需要事先登陆b站，还要手动复制粘贴cookie，有些麻烦，但好在代码量很少，速度也很快。

    后续开发

    在方案一的基础上，又针对某些可以点歌的直播间设增加了点歌的功能，虽然她们唱的不咋好听吧。思路是爬取网易云歌单中的歌曲，保存到列表中，然后将歌曲名按照点歌的格式
    发送到直播间，并且设置一定的点歌间隔。网易云的反爬措施做的还可以，歌曲时长的获取有点麻烦，因此没有后续的更新打算。就算获取到时长也很有可能遇到其他人插队点歌，看起来
    还需要增加弹幕爬虫和解析。
    至于方案二，增加了上文所说的弹幕爬虫和解析，并且将获取的弹幕转发给聊天机器人，再将聊天机器人的文本回复作为弹幕发送。挺无聊的一个小功能，因为机器人都太蠢了……

    总结

    一个好的爬虫方案的实现，需要耐心和不断的测试。浏览器，正则表达式，加密解密等知识都要掌握，至少知道方向在哪儿。
